# Sesión 11 - Práctica: Relaciones entre Clases

## Objetivo

Aplicar los conceptos de relaciones entre clases aprendidos en las sesiones anteriores, con énfasis en visibilidad y control de acceso.

## Contenidos

### 1. Visibilidad y modificadores de acceso

Práctica sobre el uso correcto de modificadores de acceso (`public`, `private`, `protected`) en el contexto de relaciones entre clases.

**Ejemplo**: [visibilidades-public-private-protected](./visibilidades-public-private-protected/)

Este ejemplo muestra:
- Diferencias entre modificadores de acceso
- Cómo controlar el acceso a atributos y métodos
- Encapsulamiento en relaciones entre objetos
- Casos de uso para cada nivel de visibilidad

### 2. Práctica con relaciones

**Ejemplo**: [Imagine Pidgeons](./Imagine%20Pidgeons/)

Ejercicio práctico que integra:
- Asociaciones entre clases
- Composición y agregación
- Control de visibilidad
- Navegabilidad entre objetos relacionados

## Conceptos clave

- **Modificadores de acceso**: Control del alcance de atributos y métodos
- **Encapsulamiento**: Ocultar detalles de implementación
- **Relaciones**: Implementación práctica de asociaciones, composición y agregación
- **Visibilidad**: Determinar qué elementos son accesibles desde otras clases
